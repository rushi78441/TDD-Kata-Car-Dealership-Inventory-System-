## 📋 Session Log & Prompts

### Phase 1: Architecture & Backend TDD

#### Prompt 1: Initial Backend Setup & TDD Strategy
> **User Prompt:**
> "I need to build a RESTful FastAPI backend for a Car Dealership Inventory System using TDD. The database should be PostgreSQL with SQLAlchemy 2.0 (Async). We need JWT authentication (Register/Login) and vehicle routes (`GET /api/vehicles`, `GET /api/vehicles/search`, `POST /api/vehicles`, `PUT /api/vehicles/:id`, `DELETE /api/vehicles/:id`, `/purchase`, `/restock`). Please generate Pytest unit test cases first following the Red-Green-Refactor approach."

* **Outcome:** Generated `test_auth.py` and `test_vehicles.py` using `pytest-asyncio` and `httpx.AsyncClient`. Implemented async models and Pydantic DTO schemas to pass tests.

> **User Prompt:**
> "How can I improve my Test Case Logic, which are the cases I must Assert in tests"

* **Outcome:** Generated Some Test asssertion cases like asserting negative stock number, zero value assertion in restock api tests.

### Phase 2: Frontend Architecture & UI Optimization

#### Prompt 2: Optimistic UI Updates for Inventory Actions
> **User Prompt:**
> "Generate UI Boilerplate components for Login Page, Registration page, and Admin Page, ,vivid Components UI (Html,tailwind) boilerplates"

* **Outcome:** Generated Elegent UI pages and components.


> **User Prompt:**
> "In my React (Vite) + Tailwind frontend, clicking the Purchase, Restock, or Delete buttons causes a full page refresh/re-fetch from Supabase, making the app feel slow. Please optimize `AdminPage.jsx` and `VehicleCard.jsx` to perform **Optimistic UI Updates** so state changes instantly on click, and handle backend state rollbacks gracefully on error."

* **Outcome:** Removed full table re-fetches (`loadVehicles()`). Implemented immediate React state mutation with automatic rollbacks on network failure.

---

### Phase 3: Deployment, Database Tuning & Docker

#### Prompt 3: Supabase PgBouncer & Vercel Connection Issues
> **User Prompt:**
> "When connecting my FastAPI app deployed on Vercel to Supabase PostgreSQL, requests take too long or time out due to serverless cold starts and direct port 5432 overhead. How can I optimize SQLAlchemy async connection parameters with Supabase's transaction pooler on port 6543?"

* **Outcome:** Configured `DATABASE_URL` to use port `6543` (`.pooler.supabase.com`) and updated `engine.py` with `prepared_statement_cache_size=0` to prevent statement prepared conflicts over PgBouncer. And Generated Code in frontend that Gives smooth instant UI Updation instead of waiting for DB Update in cloud.

#### Prompt 4: Containerization & CI Workflow
> **User Prompt:**
> "Write multi-stage Dockerfiles for both backend (Python 3.12 slim) and frontend (Node build -> Nginx alpine), along with a GitHub Actions workflow `.github/workflows/ci.yml` that runs pytest and frontend build verification on push to main."

* **Outcome:** Generated production-grade container specifications and continuous integration pipelines.

> **User Prompt:**
> "Check Codebase Why CI workflow is failing , Help me debug this errors"

* **Outcome:** Analyzed and give me response with some python circular import errors, and one non tracked file in git, helped me to fix and resolve errors and make my CI Workflow green.

## 🧠 Reflection on AI Usage

1. **Efficiency:** Using AI assistants allowed focusing on higher-order system design (Async state orchestration, rate-limiting algorithms, and database pooling) rather than typing out repetitive boilerplate.
2. **TDD Workflow:** Prompting for tests prior to backend endpoint implementation forced clear boundary definitions and schema structures early in the development lifecycle.
3. **Responsibility & Verification:** Every block of code generated by AI was manually reviewed, executed against unit test suites, and adjusted to meet exact Incubyte kata requirements.
